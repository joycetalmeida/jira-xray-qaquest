## 2. Scope / Escopo

This policy applies to data processed by the QAQuest Forge app in Jira Cloud environments where the app is installed.

Esta política se aplica aos dados processados pelo app QAQuest (Forge) em ambientes Jira Cloud onde o app está instalado.

---

## 3. Data We Process / Dados que Processamos

The App processes only data required to provide gamification and reporting features in Jira/Xray contexts, such as:

- Jira account identifiers (for example, accountId)
- Issue metadata and content needed for scoring (for example: issue key, issue type, summary, description, attachments metadata, comments metadata, status and changelog data)
- Project identifiers (for example: project key or project id)
- Computed gamification outputs (for example: points, level, badges, leaderboard/report metrics)
- Technical operational logs (for example: event type, issue key, user account id, points assigned)

O App processa apenas os dados necessários para fornecer funcionalidades de gamificação e relatórios no Jira/Xray, incluindo:

- Identificadores de conta do Jira (por exemplo, accountId)
- Metadados e conteúdo de issue necessários para pontuação (por exemplo: chave da issue, tipo, resumo, descrição, metadados de anexos, metadados de comentários, status e dados de changelog)
- Identificadores de projeto (por exemplo: key ou id do projeto)
- Saídas calculadas de gamificação (por exemplo: pontos, nível, badges, métricas de ranking/relatório)
- Logs técnicos operacionais (por exemplo: tipo de evento, chave da issue, account id do usuário, pontos atribuídos)

## 4. Purpose and Legal Basis / Finalidade e Base Legal

Purpose:

- Provide gamification scoring for Test and Test Execution workflows
- Build project and personal quality/performance reports
- Show user feedback notifications related to scoring actions
- Ensure app security, reliability and troubleshooting

Finalidade:

- Fornecer pontuação de gamificação para fluxos de Test e Test Execution
- Construir relatórios pessoais e de projeto
- Exibir notificações de feedback relacionadas a ações de pontuação
- Garantir segurança, confiabilidade e diagnóstico do app

Legal basis typically relies on legitimate interests and/or contract performance of the customer organization using Jira Cloud, according to applicable law.

A base legal normalmente se apoia em legítimo interesse e/ou execução contratual da organização cliente que utiliza Jira Cloud, conforme legislação aplicável.

## 5. Data Sources / Fontes dos Dados

- Data is obtained from Jira Cloud APIs and Forge event payloads triggered by Jira issue events.
- Dados são obtidos por APIs do Jira Cloud e payloads de eventos Forge disparados por eventos de issue no Jira.

---

## 6. Storage and Retention / Armazenamento e Retenção

Current app behavior:

- The current version does not persist personal data in an external database by default.
- Processing is primarily event-based and report-based over Jira data.
- Operational logs may contain minimal identifiers required for debugging and are retained only for the period necessary for operational support and security.

Comportamento atual do app:

- A versão atual não persiste dados pessoais em banco externo por padrão.
- O processamento é majoritariamente baseado em eventos e relatórios sobre dados do Jira.
- Logs operacionais podem conter identificadores mínimos para diagnóstico e são mantidos apenas pelo período necessário para suporte operacional e segurança.

If a future version enables external storage, this policy will be updated before that processing becomes active.

Se uma versão futura habilitar armazenamento externo, esta política será atualizada antes da ativação desse processamento.

## 7. Sharing and International Transfers / Compartilhamento e Transferência Internacional

- We do not sell personal data.
- We do not share personal data with third parties except when required to operate Atlassian Forge/Jira Cloud services, comply with law, or protect legal rights.
- International transfers, when applicable, follow Atlassian platform and applicable legal safeguards.

- Não vendemos dados pessoais.
- Não compartilhamos dados pessoais com terceiros, exceto quando necessário para operar os serviços Atlassian Forge/Jira Cloud, cumprir obrigação legal ou proteger direitos legais.
- Transferências internacionais, quando aplicáveis, seguem a plataforma Atlassian e as salvaguardas legais cabíveis.

## 8. Security / Segurança

We apply technical and organizational measures proportionate to risk, including:

- Atlassian Forge permission scoping
- Access controls based on Atlassian account context
- Minimal data processing aligned to feature needs
- Monitoring and logging for incident investigation

Aplicamos medidas técnicas e organizacionais proporcionais ao risco, incluindo:

- Escopos de permissão no Atlassian Forge
- Controle de acesso baseado no contexto da conta Atlassian
- Processamento mínimo de dados alinhado às funcionalidades
- Monitoramento e logs para investigação de incidentes
